C: Quick Picks

Trending/suggested stocks as clickable cards. Looks nice but requires wiring click → chat message injection (bidirectional state sync). More work for a landing page that users only see once.

Complexity Warning

Quick picks require the dashboard to inject messages into the chat when clicked (e.g., "Research AAPL for me"). This means:

  • Dashboard needs write access to chat state (not just read)
  • Chat needs to trigger dashboard updates when agent responds
  • Two-way binding between UI layers — biggest source of bugs in chat+dashboard apps

Recommendation: Start with Option A (prompt-first) — the chat panel IS the entry point. Add quick picks later as a progressive enhancement once the state management is solid.
